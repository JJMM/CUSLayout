CUSLayout

Auto layout for iOS.Simlar to Android,SWT,SWING API.It solves the problem of iPhone, iPhone5, iPad, resolution, rotation etc.

Download this project as a .zip file Download this project as a tar.gz file

CUSLayout为iOS下提供托管定位机制，系统提供的绝对定位方式极不方便使用，另外iPhone的4寸屏幕的出现和iPad令iOS开发者在布局需要花更多的时间，然而iOS6.0提供的AutoLayout机制令人失望，所以笔者参考Android、SWT、SWING等布局机制，编写了适合iOS下使用的CUSLayout，使用CUSLayout有以下几个好处：

  • 简化编码，不需要考虑到像素级别，仅针对区域性编程，极大的提高编程效率
  • 良好的可读性，通过布局类型即可初步了解布局意图和子控件大致摆放方式，省去了令人繁琐的还原坐标的步骤
  • 在UIView容器翻转、大小变化、支持多种分辨率时，自动处理
  • CUSLayou基于UIView布局，可完美迁移到任何应用种，不会影响原有应用
  • API简单易用，学习成本很低，通过样例程序，即可初步掌握
    心动不如行动，搞起！ # How to add CUSLayout to your project The easiest way to do add CUSLayout to your existing project is to add it as a git SubModule, and then import the project as part of yours:

In Xcode:

1、open YOUR existing project (or create a new one) drag and drop the CUSLayout.xcodeproj file downloaded from github previously from Finder to your project (either root or under Frameworks)

2、In YOUR project configuration: in the Build Phases, Add CUSLayout (the lib, not the example app) as a Target Dependency in the Link Binary With Libraries section, add the libCUSLayout.a library In YOUR Prefix.pch file, add:

3、#import

In YOUR project configuration, on the “Build Settings” tab

4、locate the “User Header Search Paths” setting, and set the Release value to "${PROJECT_DIR}/CUSLayout" (including quotes!) and check the “Recursive” check box. The Debug value should already be set, but if it’s not, change that as well. 5、Also locate the “Always Search User Paths” value and set it to YES. That should be all there is. Change the build target to “iOS Device” , and run Product->Archive. Good luck!
